# Модуль Objects — Руководство пользователя

Модуль **Objects** — центральный редактор базы данных времени выполнения для osysHome. Он управляет всей объектно-ориентированной моделью данных системы: классами, объектами, свойствами, методами, расписаниями и правами доступа.

---

## Что такое модуль Objects?

Модуль Objects предоставляет визуальный интерфейс для:

- Определения **Классов** — шаблонов, описывающих, какие свойства и методы имеет группа объектов
- Создания **Объектов** — именованных экземпляров, хранящих значения свойств и выполняющих методы
- Установки **значений свойств** в реальном времени через WebSocket
- Запуска **Методов** (Python-скриптов) с немедленным отображением результата
- Планирования автоматических действий через cron или разовые задачи
- Управления **Правами доступа** для каждого класса или объекта
- **Экспорта / Импорта** всей модели данных в формате JSON

Другие плагины (z2m, Tuya, MQTT и др.) автоматически создают и обновляют объекты — модуль Objects позволяет их просматривать и редактировать.

---

## Обзор интерфейса

Перейдите в **Admin → Objects** для открытия главного списка.

### Панель инструментов

| Кнопка | Действие |
|---|---|
| **Добавить класс** | Открывает диалог создания нового класса |
| **Добавить объект** | Открывает диалог создания отдельного объекта |
| **Права доступа (глобальные)** | Редактор глобальных прав доступа |
| **Настройки** | Переключение "Показывать ID в списках" и "Рендерить шаблоны в списке" |
| **Компактный вид** | Уменьшает визуальную плотность дерева и скрывает второстепенные описания/превью |
| **Развернуть все** | Разворачивает загруженное дерево классов |
| **Свернуть все** | Сворачивает все видимые ветки |
| **Экспортировать всё** | Скачать полное дерево классов/объектов в JSON |
| **Импорт из файла** | Загрузить JSON-файл для восстановления или расширения модели |
| Поле фильтрации | Живой поиск по именам и описаниям классов/объектов |

Панель инструментов закреплена сверху, поэтому поиск и действия с деревом остаются доступны при прокрутке.

### Дерево классов

Классы отображаются в виде раскрывающегося аккордеона. Каждая карточка показывает:

- Имя и описание класса
- Количество дочерних классов и объектов
- Основную кнопку **Изменить**
- Меню **Другие действия** для перехода к Свойствам, Методам, Объектам, Шаблону, Правам доступа и Удалению

Ветки дерева загружаются лениво при раскрытии, поэтому начальная загрузка страницы остаётся быстрой даже при большой объектной модели.

Нажмите на имя/строку класса для раскрытия или сворачивания. Состояние сохраняется в `localStorage`.

### Отдельные объекты

Объекты, не привязанные ни к одному классу, отображаются в отдельном блоке **Объекты без класса** под деревом классов.

Карточка объекта показывает:

- Имя объекта и описание
- Необязательное превью шаблона (если включено в настройках)
- Основную кнопку **Изменить**
- Меню **Другие действия** для перехода к Свойствам, Методам, Шаблону, Структуре, Расписаниям, Правам и Удалению

## Классы

### Создание класса

1. Нажмите **Добавить класс** на панели инструментов.
2. Заполните:
   - **Имя** — уникальный идентификатор без пробелов и точек (например: `Sensor`, `Light`)
   - **Описание** — необязательная метка
   - **Родительский класс** — выберите для наследования свойств и методов
3. Нажмите **Сохранить**.

### Редактирование класса

Нажмите на иконку **карандаша** в карточке класса или перейдите по `Objects?view=class&class=<id>`. Страница класса имеет вкладки:

- **Общие** — имя, описание, родительский класс
- **Свойства** — список собственных и унаследованных свойств
- **Методы** — список собственных и унаследованных методов
- **Объекты** — все объекты данного класса
- **Шаблон** — Jinja2 HTML-шаблон для отображения объектов на Dashboard
- **Права** — управление доступом
- **Инструменты** — экспорт класса в JSON

В верхней части страницы класса также показывается компактный summary-блок:

- ID класса
- Общее количество свойств (собственные + унаследованные)
- Общее количество методов (собственные + унаследованные)
- Количество объектов этого класса

Кнопки `+` для добавления свойств, методов и объектов визуально привязаны к соответствующим вкладкам.

### Удаление класса

Нажмите **Удалить** в карточке класса (отображается только если класс не имеет дочерних классов и объектов).

## Объекты

### Создание объекта

1. Нажмите **Добавить объект** на панели (или нажмите **+** внутри карточки класса для предварительной привязки).
2. Заполните:
   - **Имя** — уникальный идентификатор без пробелов и точек (например: `BedroomSensor`, `KitchenLight`)
   - **Описание** — необязательная метка
   - **Класс** — выберите шаблон класса (не обязательно для отдельных объектов)
3. Нажмите **Сохранить**. Вы будете перенаправлены на страницу объекта для добавления свойств.

### Страница объекта (вкладки)

Перейдите по `Objects?view=object&object=<id>` для открытия объекта. Страница имеет:

#### Вкладка «Общие»

Изменить имя, описание и привязку к классу.

В верхней части страницы объекта также показывается компактный summary-блок:

- ID объекта
- Количество свойств
- Количество методов
- Количество расписаний
- Назначенный класс

#### Вкладка «Свойства»

Список всех свойств объекта, включая унаследованные от класса. Каждая строка показывает:

- Имя свойства (`ИмяОбъекта.имяСвойства`)
- Описание, связанные свойства, привязанный метод, задачи по расписанию
- Текущее **значение** — обновляется в реальном времени через WebSocket
- Кнопка **Изменить значение** (иконка карандаша) — открывает модальный редактор

**Редактор значений** поддерживает 7 типов:

| Тип | Редактор |
|---|---|
| `int` | Числовой ввод с подсказками min/max/step |
| `float` | Десятичный ввод с шагом и точностью |
| `str` | Текстовая область или выпадающий список (если задан `allowed_values`) |
| `bool` | Переключатель (True / False) |
| `datetime` | Выбор даты и времени |
| `enum` | Выпадающий список с метками из `enum_values` |
| `dict` / `list` | JSON-редактор |

Изменения отправляются немедленно через WebSocket — перезагрузка страницы не требуется.

Кнопки действий для свойств:

- **Изменить** — перейти к форме свойства для редактирования типа/валидации
- **Добавить задачу** — добавить задачу по расписанию для этого свойства
- **Удалить** — удалить свойство
- **Переопределить** — для унаследованных свойств: создать переопределение на уровне объекта

#### Вкладка «Методы»

Список всех методов. Каждая строка показывает время последнего выполнения, источник и результат.

- **Запустить** — выполнить метод немедленно; результат отображается на месте
- **Изменить / Удалить** — для методов уровня объекта
- **Переопределить** — для унаследованных методов класса
- **Добавить задачу** — запланировать периодическое выполнение

#### Вкладка «Шаблон»

Редактирование Jinja2 HTML-шаблона для отображения объекта на Dashboard. Шаблоны родительских классов показаны ниже для справки.

#### Вкладка «Структура»

Полная структура объекта (свойства, методы, родители, права) с обновлением значений через WebSocket.

#### Вкладка «Расписания»

Список всех запланированных задач. Вы можете:

- **Включить / Отключить** — без перезагрузки страницы
- **Изменить** — перейти к форме редактирования расписания
- **Удалить** — немедленно удалить задачу

#### Вкладка «Права»

Настройка прав чтения/записи по группам пользователей.

#### Вкладка «Инструменты»

- **Экспортировать объект** — скачать в JSON
- **Клонировать объект** — создать копию со всеми свойствами и значениями

## Свойства

### Добавление свойства

Перейдите к объекту или классу, откройте вкладку **Свойства** и нажмите **+**.

| Поле | Описание |
|---|---|
| **Имя** | Без пробелов и точек. Пример: `temperature`, `is_on` |
| **Описание** | Необязательная метка |
| **Метод** | Привязка к методу, вычисляющему значение свойства |
| **История (дней)** | Сколько дней хранить историю изменений (0 = отключено) |
| **Тип** | `bool`, `int`, `float`, `str`, `datetime`, `dict`, `list`, `enum` |
| **Иконка** | Класс FontAwesome (`fas fa-thermometer`) или Iconify (`mdi:thermometer`) |
| **Цвет** | Акцентный цвет строки свойства |
| **Порядок сортировки** | Порядок отображения (меньше = выше) |
| **Значение по умолчанию** | Показывается при отсутствии значения |
| **Только чтение** | Запрещает редактирование через интерфейс |

**Валидация по типу:**

- `int`/`float`: минимум, максимум, шаг
- `float`: количество знаков после запятой
- `str`: паттерн регулярного выражения
- `enum`: JSON-словарь ключ → метка (`{"0": "Выкл", "1": "Вкл"}`)

**Расширенная валидация:**

- **Допустимые значения** — JSON-массив ограничений (`[1, 5, 10, 20]`)
- **Ограничение частоты** — минимальный интервал между изменениями (секунды)
- **Зависимости** — условная валидация на основе другого свойства

## Методы

### Добавление метода

Перейдите к объекту или классу → вкладка **Методы** → **+**.

| Поле | Описание |
|---|---|
| **Имя** | Идентификатор без пробелов и точек |
| **Описание** | Необязательное |
| **Код** | Python-код, выполняемый в среде osysHome |
| **Вызов родителя** | До / после / никогда — относительно метода родительского класса |
| **Периодичность** | Настройка cron-расписания прямо в форме метода |

Код пишется в встроенном редакторе Monaco/Ace. Сохранение **асинхронное** — страница не перезагружается; успех/ошибка отображаются на месте.

---

## Расписания

Задачи по расписанию привязаны к объектам.

| Поле | Описание |
|---|---|
| **Имя** | Генерируется автоматически, можно изменить |
| **Код** | Python-выражение (например: `setProperty("Obj.prop", 1)`) |
| **Crontab** | Выражение cron (например: `*/5 * * * *`) |
| **Время запуска** | Одноразовое выполнение |
| **Активно** | Включить/отключить без удаления |

---

## Права доступа

Вкладка **Права** (для класса или объекта) управляет:

- **Чтение** — кто может читать значения свойств
- **Запись** — кто может устанавливать значения свойств
- **Выполнение** — кто может запускать методы

Глобальные права: `Objects?view=permissions`

## Импорт / Экспорт

### Экспорт

- **Экспортировать всё** — полный дамп: `GET /api/export/all`
- **Экспортировать класс** — один класс с объектами и дочерними классами
- **Экспортировать объект** — один объект с свойствами и значениями

### Импорт

Нажмите **Импорт из файла** на панели инструментов:

1. Выберите `.json` файл
2. Настройте опции:
   - **Добавить классы** — импортировать определения классов
   - **Добавить объекты** — импортировать данные объектов
   - **Перезаписать** — заменить существующие классы/объекты с тем же именем
3. Нажмите **Загрузить**

## Обновления в реальном времени через WebSocket

Модуль Objects подключён к WebSocket-серверу osysHome. Когда любой плагин вызывает `setProperty("Объект.свойство", значение)`, новое значение появляется **мгновенно** во вкладке Свойства — без перезагрузки.

Редактор значений также использует WebSocket для доставки нового значения и отображает ошибки валидации бэкенда прямо в модальном окне.

## Советы

- **Соглашения об именах**: Используйте `PascalCase` для классов (`SensorClass`) и `CamelCase` или `snake_case` для объектов (`BedroomSensor`, `kitchen_sensor`).
- **Шаблоны**: Используйте систему Jinja2 для отображения значений свойств как HTML-виджетов на Dashboard.
- **Наследование**: Размещайте общие свойства/методы в классе — все объекты класса автоматически их получают.
- **Свойства только для чтения**: Помечайте свойства как `read_only`, чтобы предотвратить случайные изменения через UI (плагины всё равно могут обновлять их через `setProperty`).
- **Большие деревья**: Сначала используйте фильтр, а `Развернуть все` включайте только когда действительно нужно просмотреть всю иерархию сразу.
