# Objects - Модуль управления объектами

![Objects Icon](static/objects.png)

Комплексная объектно-ориентированная система управления для создания, редактирования и управления классами, объектами, свойствами и методами с расширенными возможностями валидации.

## Описание

Модуль `Objects` предоставляет полнофункциональную систему управления объектами для платформы osysHome. Он позволяет создавать иерархические структуры классов, создавать экземпляры объектов из классов, определять свойства с расширенными правилами валидации, создавать методы и управлять жизненным циклом объектов через планирование и права доступа.

## Основные возможности

- ✅ **Управление классами**: Создание иерархических структур классов с наследованием
- ✅ **Управление объектами**: Создание и управление экземплярами объектов из классов
- ✅ **Управление свойствами**: Определение свойств с расширенной валидацией:
  - Валидация типов (int, float, str, bool, enum и др.)
  - Валидация диапазона (min/max)
  - Валидация шага (дискретность)
  - Разрешенные значения
  - Ограничение частоты изменений
  - Зависимости между свойствами
  - Регулярные выражения
- ✅ **Управление методами**: Определение и выполнение методов на объектах и классах
- ✅ **Планирование**: Планирование операций с объектами и вызовов методов
- ✅ **Права доступа**: Управление контролем доступа для объектов и классов
- ✅ **Интеграция поиска**: Полнотекстовый поиск по классам, объектам, свойствам, методам и значениям
- ✅ **Поддержка виджетов**: Виджет панели управления с статистикой объектов
- ✅ **Рендеринг шаблонов**: Настраиваемые шаблоны объектов

## Структура модуля

### Классы
Классы определяют структуру и поведение для объектов. Они поддерживают:
- Иерархическое наследование (родительско-дочерние отношения)
- Определение свойств
- Определение методов
- Описание и метаданные

### Объекты
Объекты являются экземплярами классов (или автономными объектами). Они имеют:
- Свойства со значениями
- Методы, которые можно выполнять
- Шаблоны для рендеринга
- Возможности планирования
- Права доступа

### Свойства
Свойства определяют атрибуты объектов и классов:
- **Типы**: int, float, str, bool, enum, date, time, datetime и др.
- **Валидация**: min, max, step, allowed_values, regexp
- **Расширенные возможности**: rate_limit, depends_on
- **Метаданные**: icon, color, description

### Методы
Методы определяют действия, которые можно выполнять над объектами:
- Выполнение кода
- Поддержка параметров
- Планирование
- Права доступа

## Панель администратора

Модуль предоставляет комплексный административный интерфейс, доступный через главную панель администратора:

### Главный вид
- **Дерево классов**: Иерархический вид всех классов
- **Автономные объекты**: Объекты, не принадлежащие ни одному классу
- **Настройки**: Параметры конфигурации модуля

### Управление классами (`view=class`)
- Создание, редактирование и удаление классов
- Управление иерархией классов
- Определение свойств и методов класса
- Просмотр информации о классе

### Управление объектами (`view=object`)
- Создание, редактирование и удаление объектов
- Назначение объектов классам
- Управление свойствами и значениями объектов
- Определение методов объектов
- Настройка шаблонов
- Планирование операций

### Управление свойствами (`view=property`)
- Создание и редактирование свойств
- Настройка правил валидации
- Настройка зависимостей
- Определение метаданных (icon, color)

### Управление методами (`view=method`)
- Создание и редактирование методов
- Написание кода методов
- Настройка параметров
- Установка прав доступа

### Планирование (`view=schedule`)
- Планирование операций с объектами
- Настройка повторяющихся задач
- Управление запланированными заданиями

### Права доступа (`view=permissions`)
- Управление контролем доступа
- Настройка прав пользователей
- Установка прав на уровне объектов

## Расширенная валидация

Модуль поддерживает расширенные функции валидации для свойств. Подробную документацию см. в [VALIDATION_GUIDE.md](VALIDATION_GUIDE.md).

### Типы валидации

1. **Step (Дискретность)**: Для типов int и float, определяет шаг изменения значений
2. **Allowed Values (Разрешенные значения)**: Ограничивает значения свойств конкретным списком
3. **Rate Limit (Ограничение частоты)**: Предотвращает изменения свойств чаще указанного интервала
4. **Depends On (Зависимости)**: Определяет зависимости от значений других свойств

### Пример: Умный термостат

```json
{
  "type": "float",
  "params": {
    "min": 18.0,
    "max": 30.0,
    "step": 0.5,
    "decimals": 1,
    "rate_limit": 10.0,
    "depends_on": {
      "property": "enabled",
      "value": true,
      "condition": "equals"
    }
  }
}
```

## Использование

### Доступ к модулю

Перейдите в панель администратора и выберите модуль "Objects" из категории System.

### Создание класса

1. Нажмите "Add Class" или выберите существующий класс
2. Введите имя и описание класса
3. При необходимости выберите родительский класс для наследования
4. Сохраните класс

### Создание объекта

1. Нажмите "Add Object" или выберите класс для создания объекта
2. Введите имя и описание объекта
3. При необходимости выберите класс
4. Настройте шаблон при необходимости
5. Сохраните объект

### Добавление свойств

1. Перейдите к объекту или классу
2. Откройте вкладку "Properties"
3. Нажмите "Add Property"
4. Настройте:
   - Имя и тип
   - Параметры валидации (min, max, step и т.д.)
   - Расширенную валидацию (allowed_values, rate_limit, depends_on)
   - Метаданные (icon, color)
5. Сохраните свойство

### Добавление методов

1. Перейдите к объекту или классу
2. Откройте вкладку "Methods"
3. Нажмите "Add Method"
4. Введите имя метода и код
5. Настройте параметры и права доступа
6. Сохраните метод

## Интеграция поиска

Модуль предоставляет функциональность поиска, которая индексирует:
- Классы (по имени и описанию)
- Объекты (по имени и описанию)
- Свойства (по имени и описанию)
- Методы (по имени, описанию и коду)
- Значения (по содержимому значения)

Результаты поиска включают прямые ссылки на соответствующие элементы.

## Виджет

Модуль предоставляет виджет панели управления, показывающий:
- Общее количество классов
- Общее количество объектов
- Общее количество значений свойств

## Конфигурация

Настройки модуля, доступные в панели администратора:

- **Render Templates**: Включить/выключить рендеринг шаблонов в списке объектов
- **Show ID**: Отображать ID объектов/классов в интерфейсе

## Функциональность перезагрузки

Модуль поддерживает перезагрузку объектов из хранилища:

- **Перезагрузить класс**: `Objects?view=reload&type=class&id={class_id}`
- **Перезагрузить объект**: `Objects?view=reload&type=object&id={object_id}`

## Технические детали

- **Модели базы данных**: Class, Object, Property, Method, Value
- **Хранилище**: Интегрировано с ObjectsStorage для управления объектами во время выполнения
- **Права доступа**: Контроль доступа на основе ролей (admin/root видят все, остальные видят несистемные элементы)
- **Шаблоны**: Поддержка шаблонов Jinja2 для рендеринга объектов
